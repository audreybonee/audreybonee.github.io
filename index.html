GenAI & Knowledge Graphs

RepoChat: GraphRAG Assistant

Standard RAG systems treat code as flat text, often missing the structural context needed for complex engineering queries. I built a Graph Retrieval-Augmented Generation (GraphRAG) engine that combats developer decision fatigue by mapping repositories into a Knowledge Graph.

Architecture: The system ingests codebases to construct nodes (functions, classes) and edges (imports, calls) in Neo4j. It combines vector search (semantic understanding) with graph traversal to answer high-level impact analysis questions, such as "If I modify the UserAuth class, which downstream services will break?"
